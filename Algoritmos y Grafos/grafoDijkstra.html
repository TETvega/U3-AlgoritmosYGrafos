¿Que es el Grafo de Dijkstra?

El teorema de Dijkstra es un concepto fundamental en el campo de la teoría de grafos y la programación de algoritmos. Fue propuesto por el científico de la computación Edsger Dijkstra en 1956 y desde entonces ha sido ampliamente utilizado en diversas aplicaciones, como la búsqueda del camino más corto en redes de transporte, la planificación de rutas en sistemas de navegación y la optimización de recursos en sistemas de distribución. Este teorema se basa en el algoritmo de Dijkstra, el cual permite encontrar el camino más corto entre dos nodos en un grafo ponderado.




¿Cuál es el funcionamiento del algoritmo de Dijkstra?

El algoritmo de Dijkstra es como un navegador que te ayuda a encontrar la ruta más corta entre dos puntos en un mapa, pero en lugar de considerar solo la distancia física entre lugares, también tiene en cuenta los pesos asociados a las aristas (caminos) en un grafo, lo que puede representar tiempo, costo u otras métricas.


  • Paso 1: Inicialización

    Comenzamos desde un nodo de origen y asignamos una distancia inicial de 0 a este nodo. Para todos los demás nodos, asignamos una distancia inicial infinita, lo que indica que aún no hemos encontrado la mejor ruta hacia esos nodos.


  • Paso 2: Exploración de Nodos

    El algoritmo selecciona el nodo más cercano al nodo de origen que aún no ha sido visitado. Este será nuestro nodo actual.


  • Paso 3: Actualización de Distancias

    Para cada nodo vecino del nodo actual, calculamos la distancia desde el nodo de origen pasando por el nodo actual y llegando al nodo vecino. Si esta distancia es menor que la distancia actualmente conocida para ese nodo vecino, actualizamos la distancia. Esto significa que hemos encontrado una ruta más corta hacia ese nodo.


  • Paso 4: Marcar Nodos Visitados

    Marcamos el nodo actual como visitado para no volver a él en futuras iteraciones. También es importante tener en cuenta que si un nodo ya ha sido visitado y se encuentra una nueva ruta más corta hacia él, no se vuelve a visitar, ya que ya hemos encontrado la ruta más corta.


  • Paso 5: Repetición y Condiciones de Parada

    Repetimos los pasos 2 al 4 hasta que hayamos visitado todos los nodos o hasta que lleguemos al nodo de destino. Esto asegura que consideremos todas las posibles rutas y encontremos la ruta más corta.


  • Manejo de Situaciones Especiales:

    • Si encontramos un nodo que ya ha sido visitado pero a través de un camino más largo, simplemente lo ignoramos y continuamos explorando otras rutas.
    • Si encontramos un camino más corto hacia un nodo que ya hemos visitado, actualizamos la distancia conocida para ese nodo y continuamos explorando desde ese nodo, ya que podría llevarnos a rutas más cortas hacia otros nodos.



Ejemplo:

Imagina que estás planificando una ruta de entrega para una empresa de logística. El algoritmo de Dijkstra te ayudaría a encontrar la ruta más corta entre almacenes y destinos finales, considerando como la distancia y el tiempo de viaje. Si descubres un atajo que reduce el tiempo de entrega, el algoritmo lo tendrá en cuenta y ajustará la ruta en consecuencia, siempre buscando la forma más eficiente de llegar a cada destino.

Código de la función de búsqueda lineal única para JS

Codigo utilizado para JS

Recibe un numero entero y una lista de numeros enteroos: ( 3 , [4 , 6, 8, 3, 9 , 0] )

Se detiene en el momento que encuentra el numero buscado.

                                
    function busquedaLinealUnica(numeroBuscado, listaNumeros) {
        let bandera = false;
        for (let i = 0; i < listaNumeros.length; i++) {
            if (numeroBuscado === listaNumeros[i]) {
                bandera = true;
                mandarResultado(`Se encontró el número buscado: "${numeroBuscado}. En la posición #${i + 1}`);
                break;
            }
        }

        if (!bandera) {
            mandarResultado(`No se encontró el número buscado: "${numeroBuscado}" en la lista proporcionada`);
        }
    }

Código de la función de búsqueda Multiple para JS

Codigo utilizado para JS

Recibe un numero entero y una lista de numeros enteroos: ( 3 , [4 , 6, 8, 3, 9 , 0] )

Se hasta recorrer completamente toda la lista de numeros

Almacena todos los indices en un arreglo 

                                
    function busquedaLinealMultiple( numeroBuscado , listaNumeros) {
        let bandera = false
    
        let numerosEncontrados= []
        for (let i = 0; i < listaNumeros.length; i++) {
            if (numeroBuscado === listaNumeros[i]) {
                bandera=true
                numerosEncontrados.push(i)
            }
        }
    
        if (!bandera) {
            mandarResultado(`No se encontro el numero buscado: "${numeroBuscado}" en la lista proporcionada`)
        }else if(bandera){
    
            let indicesLetras = ''
            numerosEncontrados.forEach( indice => {
                indicesLetras+= indice+1 +", "
            });
            mandarResultado(`Se encontro el Numero Buscado: "${numeroBuscado}".
            \nEn la posicion #${indicesLetras}`)
            
        }
    }
