Búsqueda de Grafos por Profundidad (DFS)

La Búsqueda de Grafos por Profundidad o DFS (por sus siglas en inglés: Depth First Search) es un algoritmo que nos permite explorar y buscar información en un grafo de una manera sistemática.

Una manera de visualizar el DFS es comparándolo con el viaje de un explorador en un bosque. Imaginemos a este explorador adentrándose en un bosque denso y oscuro. El explorador sigue un camino particular tan lejos como puede, retrocediendo solo cuando se encuentra con un callejón sin salida. Es como moverse por un laberinto oscuro, donde el explorador elige un camino y sigue avanzando sin tener una visión completa del laberinto en su totalidad.

¿Cómo Funciona DFS?

  1. Comenzamos en un nodo inicial: Seleccionamos un nodo para comenzar nuestra búsqueda.
  2. Exploramos a fondo: Desde el nodo inicial, seguimos una conexión a otro nodo y luego a otro, y así sucesivamente, hasta que no hayan más conexiones que seguir.
  3. Retrocedemos cuando sea necesario: Si llegamos a un punto donde no podemos seguir más, retrocedemos al último nodo donde había otras conexiones sin explorar y continuamos desde allí.
  4. Marcamos los nodos visitados: Para asegurarnos de no visitar el mismo nodo dos veces, marcamos los nodos que ya hemos explorado.

Detalles Técnicos del Algoritmo DFS

Estructura de Datos Utilizadas

En la implementación del algoritmo DFS, se hacen uso de diversas estructuras de datos para llevar a cabo el proceso de búsqueda de manera eficiente. La estructura principal es una pila, que se utiliza para almacenar los nodos que se están explorando en un momento dado. Además, se emplea un conjunto (objeto) para llevar un registro de los nodos que ya han sido visitados durante el recorrido.

Proceso de Búsqueda

El algoritmo DFS sigue un enfoque de exploración en profundidad, comenzando desde un nodo inicial y moviéndose a través de las ramas del grafo de manera recursiva o iterativa. Cuando se explora un nodo, se revisan sus nodos adyacentes y se procede a explorarlos de manera recursiva, profundizando en el grafo hasta que se alcance un punto muerto. En ese momento, se retrocede al último nodo con conexiones no exploradas y se continúa la búsqueda desde allí.

Marcado de Nodos Visitados

Para evitar recorrer el mismo nodo más de una vez y caer en un bucle infinito, se marca cada nodo visitado durante el proceso de búsqueda. Esto se logra mediante el mantenimiento de un registro de los nodos que ya han sido explorados, puede ser con un objeto.

Complejidad Temporal y Espacial

La eficiencia del algoritmo DFS puede verse influenciada por el tamaño y la complejidad del grafo en el que se aplica. En su peor caso, la cantidad de operaciones que realiza puede ser proporcional al número total de nodos y conexiones en el grafo. Esto significa que cuanto mayor sea el número de nodos y conexiones, más tiempo puede llevar ejecutar el algoritmo. En cuanto a la complejidad espacial, el algoritmo requiere memoria adicional para almacenar la pila de nodos a explorar y el registro de nodos visitados.

Ejemplo de Implementación
                                    
    const grafo = {
        'A': ['B', 'C'],
        'B': ['A', 'D', 'E'],
        'C': ['A', 'F'],
        'D': ['B'],
        'E': ['B'],
        'F': ['C']
    };

    dfs(grafo, 'D'); // Comienza el recorrido desde el nodo 'D'


    function dfs(grafo, NodoInicial) 
    {
        let visitados = {}; // Objeto para almacenar los nodos visitados
        let pila = [NodoInicial]; // Pila para el recorrido en profundidad
        
        while (pila.length > 0) // Mientras haya nodos en la pila
        {
            let NodoActual = pila.pop(); // Sacamos el último nodo de la pila
            if (!visitados[NodoActual])  // Si el nodo actual no ha sido visitado
            {   
                // Marcamos el nodo como visitado
                visitados[NodoActual] = true; 
                
                // Mostramos el nodo actual 
                console.log(NodoActual); 
                
                // Obtenemos los nodos adyacentes al nodo actual
                let adyacentes = grafo[NodoActual]; 

                // Agregamos los nodos adyacentes a la pila (si no han sido visitados) 
                adyacentes.forEach(adyacente => {
                    if (!visitados[adyacente]) { 
                        pila.push(adyacente); 
                    }
                });
            }
        }
    }

Desventaja del DFS 😬

El algoritmo de búsqueda en profundidad (DFS) es útil para explorar grafos, pero tiene una limitación importante: no considera la distancia entre nodos. Simplemente sigue ramas del grafo hasta llegar a nodos sin salida, lo que no garantiza encontrar el camino más corto.

Por ejemplo, en un grafo determinado, el DFS puede optar por explorar una rama larga antes de explorar una rama más corta que conduzca al nodo destino. Esto puede resultar en un recorrido más largo y menos eficiente en términos de distancia.

En comparación, algoritmos como BFS o Dijkstra son más eficientes para encontrar la ruta más corta en grafos no ponderados (grafo en el que las aristas no tienen un peso asociado).

    Haz clic en la letra del nodo inicial que desees en el Grafo No dirigido. Este será el punto de partida de la búsqueda.

    Luego, haz clic en la letra del nodo final que deseas en el mismo grafo. Este será el punto de llegada de la búsqueda.

  3. Visualiza el Recorrido

    Una vez seleccionados el nodo inicial y el nodo final, el algoritmo realizará una búsqueda para encontrar el camino entre ellos.

  4. Observa los Resultados

    El algoritmo mostrará el recorrido realizado para llegar desde el nodo inicial hasta el nodo final. Si el camino no es posible, se te informará sobre esta situación.
